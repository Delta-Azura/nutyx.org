Le 02-09-2016

NuTyX 8.2-beta-4 disponible avec cards 2.1.90

Ces trois dernières semaines furent très intenses. La NuTyX 8.2 beta-4 est la dernière ligne droite avant la sortie de la première release candidate de la version 8.2.

Une nouvelle ISO 64bits MINI est proposée pour cette nouvelle version. Sa taille fait 216M.

Pensez à vérifier leur somme MD5 avant de les transférer sur clé USB, CD-RW ou DVD-RW

Nouveautées pour l'utilisateur

La taille de l'iso s'est réduite de près de 20 %. Une réécriture quasi complète du script de démarrage de l'iso en est la cause.

Une fois le disque partitionné, l'installateur vous averti si vous avez oublié de crééer une partion d'échange. Libre à vous d'en créer une, si telle est le cas, elle sera automatiquement ajoutée dans le fichier /etc/fstab. L'utilisateur ne dois pas se prèocuper de formater la partition d'échange, elle(s) sera(ont) formatée(s) automatiquement.

L'ordre des actions dans le menu principale est revu afin de suivre l'ordre logique des actions à executer. Le choix du système de fichier est désormé recommandé, l'installateur recommande btrfs ou ext4.

Une fois installée, le configurateur vous demande l'ajout d'un utilisateur, si vous avez décidé d'activer le compte root, le configurateur demandera un nouveau mot de passe et insistera sur la création d'un utilisateur. Ceci n'est pas un bug.

La commande nu est supprimée et remplacée par la commande setup-nutyx avec l'option -cu. Pour ne pas perturber les utilisateurs, un alias est disponible dans le fichier de configuration .bashrc:

Une première version systemd est proposée de façon expérimentale.

Nouveautées sous le capot

Cards permet désormais l'utilisation d'alias, cad qu'un nom de paquet peut être appelé à l'aide de son synonyme. Exemple, le paquet xorg-mesa peut également s'appeler mesa ou même Mesa.

Toutes les actions d'installation sont désormais enrégistrées dans le fichier /var/log/install.

Un alias a été créé pour remplacer la commande nu: 

alias nu='sudo setup-nutyx -cu'

Nombreux bugs des scripts init.in, setup-nutyx ont été corrigés.

La configuration du kernel a été revue, celui-ci intègre désormais les modules des cartes NVIDIA, virtualbox et la carte réseau broadcom. Si vous avez choisis btrfs ou ext4 comme système de fichiers, vous ne devez plus utilser d'initrd.

Comme promis, le compilateur gcc 6.2.0 est sortie il y a un peu plus d'une semaine. Tous les paquets sont recompilé sur la nouvelle chaîne glibc 2.25, binutils 2.27 et gcc 6.2.0 pour assurer une stabilitée maximum.

Toutes les actions d'installation et de suppression de paquets sont désormais ajoutés dans le fichier log /var/log/install.

cards permet désormais d'effacer une collection complète, une suite de paquets ou un groupe de paquets. La collection d'un paquet installé est également stocké dans la base de données.

cards stocke désormais le nom de la collection, du groupe et de la famille du paquet. Avec ces informations supplémentaires, il est donc possible d'obtenir des informations plus filtrées à l'affichage de la sortie de la commande cards list par exemple.

cards stocke également le nom des groupes et des locales disponible. Les alias sont également stockés dans la base de données.

Le script bash install-nutyx-8.2-beta-4 permet d'installer NuTyX depuis votre NuTyX 8.1 stable ou depuis n'importe quel autre distribution.
